# CS330
Project Reflection

Working on this project changed the way I approach designing software. Instead of immediately jumping into writing code, I learned the importance of planning the overall design first and then breaking larger problems into smaller, manageable pieces. When creating my 3D scene, I started by deciding on a realistic desktop workspace that included a desk lamp, monitor, keyboard, and a cup filled with pens. Once I knew what I wanted to build, I thought about which primitive shapes would best represent each object while keeping the polygon count low. The desk lamp became the most complex object because it combined cylinders, spheres, and a cone, while the monitor and keyboard relied primarily on box meshes. Throughout the project, I also learned how object placement, lighting, textures, and materials all work together to create a believable scene rather than existing as separate components. The design process I followed consisted of planning the layout, building each object individually, applying textures and lighting, and then continuously refining the scene through testing and adjustments. This process taught me that investing time in planning and making incremental improvements leads to a cleaner final product, and it is an approach I will continue to use in future software projects.

My approach to developing programs also evolved throughout this project. One of the biggest improvements was learning how valuable modular code can be. Rather than repeating similar blocks of code, I relied on reusable functions such as `SetTransformations()`, `SetShaderTexture()`, and `SetShaderMaterial()` to simplify object creation and make the program easier to read and maintain. I also organized my code by separating texture loading, material definitions, lighting configuration, and rendering into dedicated functions. Iteration became one of the most important parts of my development process. Nearly every object required multiple adjustments to its position, scale, and rotation before it looked correct in the scene. Likewise, I made several revisions to the lighting, including repositioning the point light inside the desk lamp so the illumination appeared to come from the bulb itself instead of an external source. As the milestones progressed, I found myself spending less time simply trying to make the code work and more time improving its organization, readability, and realism. By the end of the project, I had developed a much more structured and methodical approach to writing software.

This project also reinforced how computer science can help me achieve my long-term goal of becoming a software engineer. Although computational graphics is only one area of computer science, it introduced me to concepts that extend far beyond creating 3D scenes. I gained a better understanding of transformations, coordinate systems, lighting models, texture mapping, and interactive camera controls while also strengthening more general programming skills such as problem solving, debugging, modular design, and iterative development. These are skills that can be applied to nearly any area of software engineering. From an educational perspective, this project provided practical experience using mathematical concepts within programming and helped me better understand how graphics applications are built. Professionally, it demonstrated the importance of writing clean, reusable, and maintainable code while developing a complete software application from planning through final implementation. Even if I do not pursue a career specifically in graphics programming, the experience I gained throughout this project has strengthened my confidence as a programmer and given me skills that will be valuable throughout my future career in software development.
